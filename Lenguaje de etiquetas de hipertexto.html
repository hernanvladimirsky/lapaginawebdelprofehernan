Estructura basica HTML

<html>: Etiqueta principal (raiz) Indica el comienzo de la página web.

</html>: Indica el fin de la pagina web.

<head> :Cabecera, en el se indican las etiquetas no visibles en la pagina web, como el idioma, estilos de visualización (CCS).

<title>:Indica el nombre de la pestaña (importantísimo para los motores de búsqueda).

<body>:Cuerpo, contenido visible de la página web, por ejemplo algunas etiquetas que estan en el son <h1>, <p>, <img src> (imagen), <a href> (vínculos o links), etc.

Primeros elementos

<h1>: Titulo de primer o alta importancia (no significa el mas grande en tamaño).

<h2>: Titulo de menor importancia que h1 (asi sucesivamente hasta <h6>).

<p>: Inserto un párrafo.

<br>: Salto de linea (esta etiqueta no tiene cierre).

<strong>: Texto negrita (con valor semantico).

<b>: Texto negrita (sin valor semantico).

<em>: Texto cursiva (con valor sematico).

<i>: Texto cursiva (sin valor semantico).

<u>: Texto subrayado (con valor sematico).

<a>: Texto subrayado (sin valor sematico).

Elementos Sematicos

<header>
Etiqueta para un contenido introductorio, es para entender o comprender el resto del contenido. No se puede anidar.

<nav>
Se utiliza para contener los links importantes o una barras de navegacion,
Permite contener lineas relevantes o muy importantes para la pagina web. Se puede encontrar dentro de un section.

<main>
Sirve para destacar la parte central del contenido del sitio web.
No deberia ser reutilzado, algunos lectores reconocen la etiqueta main y proveen un atajo para que el usuario acceda directamente a esa informacion salteando todo el resto del contenido, gracias a los robots de los motores de busqueta.

<section>
Marca una seccion de la interfaz, se utiliza para dividir el contenido entre partes.

<article>
Permite por ejemplo leer un articulo en una seccion. En un diario si quiero leer noticias no hace falta leer todas las noticas si no vamos al articulo que nos interesa.

<footer>
Es un pie de pagina puede contener informacion como informacion de contacto, direcciones, este puede repetirse y colocarse en cualquier sitio de la interfaz pero suele colocarse al final del contenido.

¿Como insertar un vinculo?

<a href="contacto.html" title="Contacto" target="_blank"> Ir a contacto </a>

El atributo target="_blank" permite abrir en enlace en una nueva pestaña.

Introduccion a Imagenes

Insertamos una imagen con la etiqueta <img src>

<img src="Imagenes/procesador.jpg" alt="imagen no encontrada" title="procesador" height="50%" width="50%" >

El mensaje del alt aparece cuando hay un error en la ruta o el servidor donde se aloja la imagen.

El contenido del title aparece cuando posicionamos el cursor sobre la imagen, el contenido del title siempre hace referencia a la imagen, en este caso un procesador.

Id

El id permite trabajar con un selector que afecta sólo a un elemento en el HTML.
El ejemplo puntual sería que un elemento necesita ciertas propiedades, por lo tanto debo implementar esa regla de estilo de forma única.

En Html:

<p id="parrafoA">Soy el parrafo A</p>
<p id="parrafoB">Soy el parrafo B</p>

En Css:

#parrafoA {color: blue;}

Clases

Se utiliza para afectar a varios elementos ni a todos ni uno solo, solo los que pertenecen a un grupo.

En Html:

<p class="Misparrafos"> Pertenezco al grupo de parrafos Misparrafos (Soy el Parrafo 1) </p>
<p class="Misparrafos"> Pertenezco al grupo de parrafos Misparrafos (Soy el Parrafo 2) </p>
<p class="Misparrafos"> Pertenezco al grupo de parrafos Misparrafos (Soy el Parrafo 3) </p>

En Css:

.Misparrafos {color: blue;}

Restricciones de Id y clases

  • No puedo dejar espacios.
  • No puedo utilizar elementos extraños o palabras reservadas. Por ejemplo un ; o un carácter extraño como ñ , o una tilde no deberían formar parte del nombre de ID o clase.
  • Tampoco puedo utilizar números al comienzo del nombre de id. Puedo usarlos dentro del nombre (al final o entre palabras pero no al inicio).
  • La última recomendación, es siempre utilizar nombres referenciales, o que nos ayuden a identificar de qué id o clase se trata.

Anclas

Las anclas nos permiten crear una navegación entre distintas secciones de una misma página.
Para armar una navegación entre secciones, hay que crear anclas (Elementos a con el atributo Id) y un link a esa ancla (elementos a con el atributo href)

Creo ancla:

<a id="top"></a>

Creo un link que me dirige a esa ancla:

<a href="#top">Ir hacia arriba</a>

Formularios

Formularios, ¿Qué es?

¿Nunca nos registramos en una red social, nunca creamos un usuario, seleccionamos nuestro género, una contraseña, ingresamos un teléfono, un email o adjuntamos un archivo? Bueno hoy veremos el ¿como? Atentos.

Todos los elementos que contenga el formulario debe estar entre las etiquetas
<form> y </form>.

Input

Es de la mas utilizadas para formularios con ella podemos definir gran parte de los controles típicos para formularios, hay entradas de tipo:

  1. Texto
  2. Casillas de verificación
  3. Casilla de opción (Radio)
  4. Archivos
  5. Textos enmascarados (Password)
  6. Datos ocultos
  7. Botones varios

Atributos de <input/>

Name → Indica el nombre que recibirá el campo al ser enviado. Contiene texto, es para identificar lo que estoy enviando al formulario. Ej: <input name=”dni” />

Type → Define en que se convertirá el control, por defecto es text. Ej: <input name=”nombre” type=”text” />

  • Text → texto
  • Number → numeros
  • Email → correo electrónico
  • Checkbox → Casilla de verificación
  • Radio → Casilla de opción
  • File → Subir un archivo
  • Password → Texto enmascarado (contraseña)
  • Hidden → Campo Oculto (es el tiempo que tarda el usuario en llenar el form)
  • Button → Muestra boton (sin acción por defecto, luego le podemos dar una accion)
  • Submit → Boton enviar
  • Reset → Boton de limpiar

Label

Es una etiqueta que mejora la usabilidad y accesibilidad de los usuarios, una etiqueta label asocia un texto a un control, de forma que al pulsar sobre el texto se activa el control asociado, además se relaciona el control con su propósito fundamental para los usuarios que necesiten ayuda para la navegación.

Atributos del <label>

Para relacionar un label con su control de usuario se utiliza el atributo for, este atributo espera el identificador del control del formulario con el que se va a asociar.

For→Es el identificador
Ejemplo= <label for="nombre"> Ingrese su nombre </label>

Value → Es un atributo del input que muestra el texto del botón en este caso va a mostrar la palabra Enviar.

<form>
<label for="nombre_mascota"> Ingrese el nombre de su mascota </label>
<input type="text" id="nombre_mascota" />
<input type="submit" value="Enviar"/>
</form>

Id → Es un atributo propio de la etiqueta Input
