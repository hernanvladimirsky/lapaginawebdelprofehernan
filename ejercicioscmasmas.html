Ejercicio 2 - Uso básico de cin y cout

Este ejercicio te permitirá aplicar los conceptos de la estructura básica de C++ y practicar el uso de cin y cout con los operadores << y >>.

🧠 Instrucciones

  1. Escribí un programa en C++ que pida al usuario su nombre y su edad.
  2. Luego, debe mostrar un mensaje que diga: "Hola [nombre], tenés [edad] años."
  3. Guardá el archivo con el nombre saludo.cpp.
  4. Compilalo y ejecutalo para verificar que funcione correctamente.

🔍 Explicación del programa

  • #include <iostream> → Permite usar cout y cin.
  • string nombre; → Declara una variable para guardar texto.
  • int edad; → Declara una variable para guardar números enteros.
  • cout << → Muestra un mensaje al usuario.
  • cin >> → Permite ingresar datos desde el teclado.

✅ Ejemplo de ejecución

Salida en pantalla:

  Ingrese su nombre: Ana
  Ingrese su edad: 17
  Hola Ana, tenes 17 años.

📘 Sentencia - IF

El condicional if permite ejecutar un bloque de código solo si una condición se cumple (es verdadera). Si no se cumple, el bloque se salta.


Ejercicio Modelo

🧠 Instrucciones

  1. Escribí un programa que pida al usuario un número entero.
  2. Si el número es mayor o igual a 18, debe mostrar "Sos mayor de edad".
  3. Si el número es menor que 18, debe mostrar "Sos menor de edad".

🔍 Explicación del código

  • if (edad >= 18) → Comprueba si la edad ingresada es 18 o más.
  • Si la condición es verdadera, muestra el mensaje de “mayor de edad”.
  • Si no, ejecuta el bloque del else y muestra “menor de edad”.
  • Los operadores de comparación más usados son:
    • > → mayor que
    • < → menor que
    • >= → mayor o igual que
    • <= → menor o igual que
    • == → igual a
    • != → distinto de

👨‍💻 Código


  #include <iostream>
  using namespace std;
  
  int main() {
      int edad;
  
      cout << "Ingrese su edad: ";
      cin >> edad;
  
      if (edad >= 18) {
          cout << "Sos mayor de edad";
      } else {
          cout << "Sos menor de edad";
      }
  
      return 0;
  }

✅ Ejemplo de ejecución

Entrada y salida:

  Ingrese su edad: 20
  Sos mayor de edad.
      
  Ingrese su edad: 15
  Sos menor de edad.

💻 Ejercicios IF

  1. Número en rango doble

    Realizar un programa que pida un número entero e indique en qué rango se encuentra: entre 1 y 50, entre 51 y 100 o mayor a 100.

  2. Sistema de notas completo

    Realizar un programa que pida una nota (entre 0 y 10). Validar que la nota sea correcta. Luego clasificarla como Excelente, Muy bien, Regular o Desaprobado.
    Excelente de la nota 9 a la 10.
    Muy bien de la nota 7 a la 8.
    Regular de la nota 4 a la 6.
    Desaprobado de la nota 3 o menos.

  3. Mayor y menor de tres números

    Realizar un programa que pida tres números enteros distintos e indique cuál es el mayor y cuál es el menor.

  4. Calculadora básica con validación

    Realizar un programa que pida dos números y una operación matemática (+, -, *, /). Mostrar el resultado correspondiente y validar que no se pueda dividir por cero.

    Condiciones que deben utilizar:

    • op == '+'
    • op == '-'
    • op == '*'
    • op == '/' && num2 != 0
    • op == '/' && num2 == 0

  5. Descuento según edad

    Realizar un programa que pida la edad de una persona, valide que no sea negativa y muestre el descuento correspondiente según el rango de edad.
    El rango de edad es de 1 a 12, de 13 a 17, de 18 a 59, y mayor a 60.

  6. Sueldo con aumento

    Realizar un programa que pida el sueldo de un empleado, valide que no sea negativo y calcule el nuevo sueldo aplicando el aumento correspondiente.

💻 Ejercicios con WHILE e IF anidados

  1. Control de asistencia

    Registrar la asistencia de alumnos. El programa debe pedir el estado del alumno (presente o ausente). El proceso continúa hasta que se escriba "fin".

    • Si el alumno está presente, aumentar el contador de presentes.
    • Si el alumno está ausente, aumentar el contador de ausentes.
    • Al finalizar mostrar la cantidad de presentes y ausentes.

  2. Control de edades en una actividad

    Registrar edades de participantes hasta que se ingrese 0.

    • Si la edad es menor de 12 años, clasificar como niño.
    • Si la edad está entre 12 y 17 años, clasificar como adolescente.
    • Si la edad es 18 o mayor, clasificar como adulto.
    • Mostrar cuántas personas hay en cada grupo.

  3. Compras en un kiosco

    Pedir precios de productos hasta que se ingrese 0.

    • Si el precio es menor a 500, clasificar como económico.
    • Si el precio está entre 500 y 2000, clasificar como precio medio.
    • Si el precio es mayor a 2000, clasificar como producto caro.
    • Mostrar el total de la compra.

  4. Control de contraseñas

    Simular un sistema de acceso con máximo 3 intentos.

    • Si la contraseña es correcta, mostrar "Acceso permitido".
    • Si es incorrecta, informar error.
    • Si se superan los 3 intentos, bloquear acceso.

  5. Registro de calificaciones

    Ingresar notas de alumnos hasta que se ingrese una nota negativa.

    • Si la nota es 7 o mayor, aprobado.
    • Si está entre 4 y 6, recupera.
    • Si es menor a 4, desaprobado.
    • Mostrar cantidades de cada grupo.

💻 Ejercicios con DO WHILE e IF anidados

  1. Validación de edad

    Pedir la edad de una persona hasta que ingrese una edad válida.

    • Si la edad es negativa, indicar error.
    • Si es menor a 18, indicar menor de edad.
    • Si es 18 o mayor, indicar mayor de edad.

  2. Encuesta de satisfacción

    Preguntar a los clientes si el servicio fue bueno, regular o malo. El programa debe repetirse hasta que el usuario decida terminar.

    • Contar cada tipo de respuesta.
    • Mostrar resultados al finalizar.

  3. Carga de productos

    Permitir ingresar nombre y precio de productos. Repetir hasta que el usuario decida terminar.

    • Si el precio es menor a 1000, clasificar como barato.
    • Si el precio está entre 1000 y 5000, clasificar como precio medio.
    • Si el precio es mayor a 5000, clasificar como caro.

  4. Sistema de menú

    Mostrar un menú con opciones: ver saldo, retirar dinero, salir.

    • Si elige ver saldo, mostrar saldo.
    • Si elige retirar dinero, pedir monto.
    • Si elige salir, terminar programa.

  5. Registro de temperaturas

    Ingresar temperaturas hasta que el usuario decida finalizar.

    • Si la temperatura es menor a 10, clasificar como frío.
    • Si está entre 10 y 25, clima templado.
    • Si es mayor a 25, calor.
