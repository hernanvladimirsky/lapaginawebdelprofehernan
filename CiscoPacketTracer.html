Ejercicio Básico en Cisco Packet Tracer - Clase C

🎯 Objetivo

Configurar una red simple con dos computadoras conectadas a un switch, asignar direcciones IP y comprobar la conectividad usando el comando ping.

🧰 Materiales necesarios

  • 2 Computadoras (PC)
  • 1 Switch (por ejemplo, Cisco 2960)
  • 2 Cables de red directos (Copper Straight-Through)

🌐 Dirección IP

Dispositivo IP Address Subnet Mask
PC1 192.168.1.10 255.255.255.0
PC2 192.168.1.11 255.255.255.0

🔧 Pasos de configuración

  1. Agregar 2 PCs y un switch al área de trabajo.
  2. Conectar cada PC a un puerto del switch usando cables directos.
  3. En cada PC, ir a Desktop > IP Configuration y asignar las direcciones IP correspondientes.

🧪 Prueba de conectividad

Desde la consola de PC1 (Desktop > Command Prompt), ejecutar:

ping 192.168.1.11

Se deben recibir respuestas exitosas ("Reply from...").

✅ Resultado esperado

Las dos PCs deben comunicarse correctamente, demostrando una conexión funcional en una red local básica.

🎯 Objetivo

Conectar dos computadoras usando un switch, asignando direcciones IP de Clase B, y verificar la conectividad mediante el comando ping.

🧰 Materiales necesarios

  • 2 computadoras (PC)
  • 1 switch (modelo 2960)
  • 2 cables de red directos (copper straight-through)

🌐 Direccionamiento IP

Dispositivo IP Address Subnet Mask
PC1 172.16.1.10 255.255.0.0
PC2 172.16.1.11 255.255.0.0

🔧 Pasos de configuración

  1. Arrastra las dos PCs y el switch al área de trabajo.
  2. Conecta PC1 al puerto FastEthernet0/1 del switch.
  3. Conecta PC2 al puerto FastEthernet0/2 del switch.
  4. En cada PC, accede a Desktop > IP Configuration y asigna la IP correspondiente.
  5. No es necesario configurar un gateway ya que están en la misma red.

🧪 Prueba de conectividad

Desde la consola de PC1 (Desktop > Command Prompt), ejecuta:

ping 172.16.1.11

Deberías obtener respuestas exitosas.

✅ Resultado esperado

Las dos PCs deben comunicarse correctamente. El ejercicio demuestra cómo trabajar con direcciones IP de Clase B en una red simple.

🧮 2️⃣ Cálculo de subredes

Dirección base: 192.168.1.0 /24

Queremos 3 subredes → tomamos 2 bits del host → /26

  • 3 Switches
  • 6 PCs (2 por subred)

  • Cada subred se conecta a un switch.
  • Cada switch conecta sus 2 PCs.
  • Cada switch se conecta a un puerto distinto del router.
💡 Recordá que el orden y la conexión física se realizan arrastrando los dispositivos y conectándolos con las líneas de enlace.

Subred PC1 PC2 Gateway

3 192.168.1.130 192.168.1.131 192.168.1.129

  1. Clic en el router → pestaña Config → Interfaces.

Fa0/1 192.168.1.65 255.255.255.192
Fa1/0 192.168.1.129 255.255.255.192

Asegurate de que cada interfaz esté encendida (ON).

  1. En cualquier PC abrir Command Prompt.
  2. Escribir:
ping 192.168.1.70

Ejercicio: DHCP Clase C en Cisco Packet Tracer (Sin CLI)

🎯 Objetivo

Configurar una red local donde un router asigne direcciones IP automáticamente (DHCP) a dos PCs mediante interfaz gráfica.

🧰 Materiales necesarios

  • 2 Computadoras (PC)
  • 1 Switch (por ejemplo, Cisco 2960)
  • 1 Router Wireless (por ejemplo, Home Router)
  • 3 Cables de red directos (Copper Straight-Through)

🔧 Pasos de configuración (modo gráfico)

  1. Arrastrar las 2 PCs, el switch y el router al área de trabajo.
  2. Conectar:
    • PC1 a Switch (Fa0/1)
    • PC2 a Switch (Fa0/2)
    • Router al Switch (GigabitEthernet0/0 a Fa0/24, por ejemplo)
  3. Haz clic en el Router > pestaña Config > GigabitEthernet0/0:
    • Encender el puerto: marcar On
    • IP Address: 192.168.1.1
    • Subnet Mask: 255.255.255.0
  4. En el Router > ir a la pestaña GUI:
    • Pool Name: RED_LOCAL
    • Default Gateway: 192.168.1.1
    • Start IP Address: 192.168.1.100
    • Subnet Mask: 255.255.255.0
    • Maximum Number of Users: 50
    • Haz clic en Add
  5. En cada PC:
    • Desktop > IP Configuration > seleccionar DHCP
    • Verifica que reciba IP automática (ej: 192.168.1.100)

🧪 Prueba de conectividad

Desde PC1 (Command Prompt), ejecutar:

ping 192.168.1.101

(o la IP que reciba PC2 por DHCP)

✅ Resultado esperado

Ambas PCs deben recibir direcciones IP automáticamente y comunicarse exitosamente a través del switch y el router usando DHCP.

Explicación de Comandos para Configurar un Router Cisco

enable
configure terminal
interface gigabitEthernet 0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit
exit
enable
Cambia al modo privilegiado del router, que permite acceder a comandos de configuración avanzados.
configure terminal
Entra al modo de configuración global desde el cual se puede configurar interfaces, rutas, etc.
interface gigabitEthernet 0/0
Accede a la configuración de la interfaz de red GigabitEthernet 0/0, la que usaremos para conectar el router a la LAN.
ip address 192.168.1.1 255.255.255.0
Asigna una dirección IP y máscara a la interfaz seleccionada. Esta IP será la puerta de enlace para los dispositivos de la red.
no shutdown
Activa la interfaz. Por defecto, las interfaces están deshabilitadas (apagadas). Este comando la enciende.
exit
Sale de la configuración de la interfaz y vuelve al modo de configuración global.
exit
Sale del modo de configuración global y vuelve al modo privilegiado.

Paso 3: Configuración de las PCs

Desde la terminal (Command Prompt) de cada PC:

PC1:

ipconfig 192.168.1.2 255.255.255.0 192.168.1.1

PC2:

ipconfig 192.168.1.3 255.255.255.0 192.168.1.1

PC3:

ipconfig 192.168.1.4 255.255.255.0 192.168.1.1

Explicación del comando ipconfig:

  • Primer campo: dirección IP del host.
  • Segundo campo: máscara de subred.
  • Tercer campo: puerta de enlace (IP del router).

Paso 4: Verificar conectividad

Desde cualquier PC, escribe:

ping 192.168.1.1

O prueba entre PCs, por ejemplo:

ping 192.168.1.3

Red Clase C con DHCP - Cisco Packet Tracer (CLI)

Topología

  • 1 Router (Router0)
  • 1 Switch (Switch0)
  • 3 PCs (PC0, PC1, PC2)

Esquema de Red

  • Red: 192.168.10.0/24
  • Gateway: 192.168.10.1
  • Rango DHCP: 192.168.10.100 - 192.168.10.150
  • Máscara: 255.255.255.0
  • DNS: 8.8.8.8

Configuración CLI del Router

enable
configure terminal

! Configura la interfaz del router
interface gigabitEthernet 0/0
ip address 192.168.10.1 255.255.255.0
no shutdown
exit

! Configura el servicio DHCP
ip dhcp excluded-address 192.168.10.1 192.168.10.99
ip dhcp pool RED_LOCAL
network 192.168.10.0 255.255.255.0
default-router 192.168.10.1
dns-server 8.8.8.8
exit
end

! Verificación
show ip dhcp binding
show running-config

Configuración de las PCs

En cada PC:

  • Ir a Desktop > IP Configuration
  • Seleccionar la opción DHCP

Comando Explicación

configure terminal Modo de configuración global
interface gigabitEthernet 0/0 Configura la interfaz LAN del router

¿Para qué sirve clock rate 64000 en Cisco?

El comando clock rate 64000 se usa para establecer la velocidad de transmisión en una interfaz serial de un router Cisco cuando éste actúa como DCE (Data Communication Equipment).

¿Por qué es necesario?

  • Define la frecuencia del reloj para la sincronización de datos en una conexión serial.
  • Solo se configura en el extremo DCE.
  • Permite que el otro extremo (DTE) se sincronice correctamente para la transmisión.

Ejemplo de uso

Router(config-if)# clock rate 64000

¿Se pueden usar otras frecuencias?

Sí, hay múltiples velocidades disponibles según la necesidad de la red o simulación.

📊 ¿Para qué sirve cada velocidad del comando clock rate?

Estas velocidades permiten simular distintos tipos de enlaces seriales en redes WAN o laboratorios de Cisco Packet Tracer. Se definen en bits por segundo (bps).

Clock Rate Velocidad Usos comunes y simulación
1200 1.2 kbps Simulación de enlaces muy lentos (históricos o de pruebas).
2400 2.4 kbps Transmisiones antiguas o equipos legados.
4800 4.8 kbps Poco frecuente, enlaces de baja capacidad.
9600 9.6 kbps Enlaces seriales de equipos antiguos o telemetría básica.
19200 19.2 kbps Velocidad típica en módems antiguos.
38400 38.4 kbps Simulación de conexiones con restricciones medianas.
56000 56 kbps Equivalente a un canal de voz digital (DS0).
64000 64 kbps Estándar de enlaces WAN básicos, muy usado en simulaciones.
128000 128 kbps Simula enlaces tipo ISDN de dos canales (BRI).
256000 256 kbps Más ancho de banda para enlaces más rápidos.
512000 512 kbps Enlaces corporativos o pruebas de mayor velocidad.
1544000 1.544 Mbps Enlace T1 completo (velocidad estándar en EE. UU.).
2000000 2 Mbps Velocidad cercana a E1 (usada en Europa y Latinoamérica).

¿Cómo saber si soy DCE?

En Cisco Packet Tracer, el router que tiene un pequeño rótulo con “DCE” en el cable serial es el que debe tener el clock rate.

Comando de ayuda (en router real)

Router(config-if)# clock rate ?

Este comando te muestra todos los valores posibles que podés usar como velocidad.

Configurar un servidor HTTP

1. Agregar los dispositivos

  1. Abrir Cisco Packet Tracer.
  2. Desde End Devices arrastrar un Server y al menos una PC al área de trabajo.
  3. Agregar un Switch y conectar los dispositivos con cable de cobre (cable directo).

2. Configurar direcciones IP

  1. Seleccionar la PC → Desktop > IP Configuration y asignar: dirección IP, máscara de subred y gateway (si corresponde).
  2. Seleccionar el Server → Desktop > IP Configuration y asignar: dirección IP, máscara de subred y gateway.

3. Activar el servicio HTTP

  1. Seleccionar el Server → pestaña Services.
  2. Elegir HTTP en la lista de servicios.
  3. Activar el servicio (activar el interruptor On).
  4. Opcional: editar la página web que servirá el servidor desde el área de edición del servicio.

4. Probar desde la PC

  1. En la PC → Desktop > Web Browser.
  2. Ingresar la dirección IP del servidor en la barra de direcciones.
  3. Verificar que la página del servidor se cargue correctamente.

📡 Configurar un servidor DHCP

1️⃣ Agregar los dispositivos

  1. 💻 Abrir Cisco Packet Tracer.
  2. 🖥️ Arrastrar un Server, varias PCs y un Switch.
  3. 🔗 Conectar todos con cables de cobre directo.

2️⃣ Configurar el servidor

  1. ⚙️ Seleccionar el Server.
  2. 🌐 En Desktop > IP Configuration, asignar:
    • Dirección IP fija
    • Máscara de subred
    • Gateway
  3. 📑 Ir a Services > DHCP:
    • ✔️ Activar el servicio DHCP (On)
    • ➡️ Configurar el Default Gateway
    • 🟢 Definir la Start IP Address
    • 📐 Definir la Subnet Mask
    • 👥 Indicar Maximum Number of Users
    • 💾 Hacer clic en Save

3️⃣ Configurar las PCs

  1. 🖱️ Seleccionar una PC.
  2. 🌍 En Desktop > IP Configuration, elegir la opción DHCP.
  3. 📡 La PC debería recibir automáticamente una dirección IP dentro del rango configurado.

4️⃣ Verificar

  1. ⚡ Usar el comando ping para comprobar conectividad con el servidor.
  2. 👀 Revisar que cada PC reciba una dirección IP diferente.

📊 Parámetros básicos del DHCP

Start IP Address Primera dirección que se entregará a los clientes.

🌐 Configurar dos Servidores DNS y Web

📋 Resumen

Con esta configuración, las PCs podrán acceder a páginas web usando nombres de dominio en lugar de direcciones IP.

1️⃣ Agregar dispositivos

  1. 💻 Abrir Cisco Packet Tracer.
  2. 🖥️ Arrastrar un Server, una o más PCs y un Switch.
  3. 🔗 Conectar todos con cables de cobre directo.

2️⃣ Configurar el servidor

  1. ⚙️ Seleccionar el Server.
  2. 🌐 En Desktop > IP Configuration, asignar:
    • Dirección IP fija
    • Máscara de subred
    • Gateway

3️⃣ Activar el servicio Web (HTTP)

  1. 🌍 En el servidor → pestaña Services > HTTP.
  2. ✔️ Activar el servicio HTTP (On).
  3. ✏️ Opcional: editar el contenido de la página web que se mostrará.

4️⃣ Configurar el servicio DNS

  1. 🔎 En el servidor → pestaña Services > DNS.
  2. ✔️ Activar el servicio DNS (On).
  3. 📑 En la tabla de configuración:
    • ✏️ Name: escribir el dominio (ejemplo: www.mipagina.com)
    • 💻 Address: colocar la IP del servidor web
    • ➕ Hacer clic en Add

5️⃣ Configurar las PCs

  1. 🖱️ Seleccionar cada PC → Desktop > IP Configuration.
  2. 🌍 Elegir DHCP o asignar IP manualmente.
  3. 📡 En el campo DNS Server, colocar la dirección IP del servidor.

6️⃣ Probar la configuración

  1. 💻 En la PC → Desktop > Web Browser.
  2. ⌨️ Escribir el dominio configurado (ejemplo: www.mipagina.com).
  3. 👀 Verificar que se cargue la página del servidor web.

📊 Parámetros clave

⚙️ Servicio 📖 Función
HTTP Permite que el servidor aloje una página web.
DNS Traduce nombres de dominio (ejemplo: www.mipagina.com) a direcciones IP.
DNS Server (en PCs) Indica a qué servidor preguntar por los nombres de dominio.

🌐 Configuración de Servidor HTTP, DNS y E-mail

📋 Resumen

Este ejercicio integra tres servicios en distintos servidores: HTTP para páginas web, DNS para resolución de nombres de dominio y E-mail para correo electrónico.

1️⃣ Agregar dispositivos

  1. 💻 Abrir Cisco Packet Tracer.
  2. 🖥️ Arrastrar los Servidores (con HTTP, DNS y E-mail), varias PCs y un Switch.
  3. 🔗 Conectar con cables de cobre directo.

2️⃣ Configurar el servidor

  1. ⚙️ Seleccionar el Server.
  2. 🌐 En Desktop > IP Configuration, asignar:
    • Dirección IP fija
    • Máscara de subred
    • Gateway

3️⃣ Servicio Web (HTTP)

  1. 🌍 En Services > HTTP.
  2. ✔️ Activar el servicio HTTP (On).
  3. ✏️ Opcional: editar el contenido de la página web.

4️⃣ Servicio DNS

  1. 🔎 En Services > DNS.
  2. ✔️ Activar el servicio DNS (On).
  3. 📑 En la tabla:
    • Name: dominio (ejemplo: www.mipagina.com)
    • Address: la IP del servidor web
    • ➕ Hacer clic en Add

5️⃣ Servicio de E-mail

  1. 📧 En Services > EMAIL.
  2. ✔️ Activar el servicio de correo.
  3. 👥 Crear cuentas:
    • User: nombre del usuario (ejemplo: alumno1)
    • Password: clave del usuario
    • Domain: dominio de correo (ejemplo: mired.com)
    • ➕ Hacer clic en Add

6️⃣ Configurar las PCs

  1. 🖱️ En cada PC → Desktop > IP Configuration.
  2. 🌍 Asignar IP manual o usar DHCP.
  3. 📡 En el campo DNS Server, colocar la IP del servidor.

7️⃣ Configuración de correo en las PCs

  1. 📧 En la PC → Desktop > Email.
  2. Completar:
    • Account Name: usuario (ej: alumno1)
    • Email Address: cuenta (ej: alumno1@mired.com)
    • Incoming Mail Server (POP3): IP del servidor
    • Outgoing Mail Server (SMTP): IP del servidor
    • Password: contraseña del usuario

8️⃣ Prueba de los servicios

  1. 🌍 En el navegador de la PC, escribir el dominio (ej: www.mipagina.com) → debe cargar la página web.
  2. 📧 En la aplicación de correo, enviar un mail desde alumno1@mired.com a alumno2@mired.com.
  3. 👀 Revisar la bandeja de entrada en la PC del segundo usuario.

📊 Parámetros clave de los servicios

⚙️ Servicio 📖 Función
HTTP Publica páginas web.
DNS Traduce nombres de dominio a direcciones IP.
E-mail (SMTP/POP3) Permite enviar y recibir mensajes de correo electrónico.

🛠️ Actividad: Red con 4 Servidores HTTP, 1 DHCP y 1 DNS

Objetivo

Configurar una red en Cisco Packet Tracer donde las PC clientes puedan acceder a los 4 servidores web mediante nombre de dominio usando DHCP y DNS automáticamente.

Requisitos

  • 🌐4 Servidores HTTP (cada uno con una página distinta)
  • 🖥️1 Servidor DNS
  • 📡1 Servidor DHCP
  • 💻4 PC clientes
  • 🔌Switch

Instrucciones

  1. 🔧Colocar los dispositivos en el área de trabajo y conectarlos con cable directo.
  2. 📡Configurar el Servidor DHCP para asignar IP, máscara, gateway y DNS automáticamente.
  3. 🖥️Configurar el Servidor DNS con los 4 registros:
    • 🌐 www.servidor1.com → IP HTTP 1
    • 🌐 www.servidor2.com → IP HTTP 2
    • 🌐 www.servidor3.com → IP HTTP 3
    • 🌐 www.servidor4.com → IP HTTP 4
  4. 🌐Configurar cada Servidor HTTP con su página web en Services → HTTP → index.html.
  5. 💻Configurar las PC clientes en modo DHCP y verificar que reciben IP, Gateway y DNS automáticamente.
  6. 🌐Probar en cada PC que se pueda abrir cada servidor por nombre de dominio en el navegador.

Resultado esperado

Todas las PC deben poder acceder a los 4 servidores web correctamente usando los nombres de dominio. Ejemplo: www.servidor1.com, www.servidor2.com, www.servidor3.com, www.servidor4.com

🌐 Actividad: Red con servidores de respaldo

Objetivo: Configurar una red donde 3 PCs puedan acceder a servicios web, correo electrónico y resolución de nombres mediante múltiples servidores.

📡 1️⃣ Dispositivos requeridos

  • 3 PCs
  • 2 Switch
  • 2 Servidores HTTP
  • 2 Servidores DNS
  • 2 Servidores DHCP
  • 2 Servidores Email

🔌 2️⃣ Conexiones

  • Conectar todas las PCs y servidores al Switch con cables Ethernet (Copper Straight-Through).

⚙️ 3️⃣ Configuración de DHCP

Configurar ambos servidores DHCP:

  • Rango: 192.168.1.100 – 192.168.1.150
  • Gateway: 192.168.1.1
  • DNS primario: 192.168.1.20
  • DNS secundario: 192.168.1.30

🌍 4️⃣ Configuración de Servidores DNS

Cargar los siguientes nombres:

  • Servidor DNS 1:
    www.sitio1.com → 192.168.1.40
  • Servidor DNS 2:
    www.sitio2.com → 192.168.1.50

🖥️ 5️⃣ Configuración de Servidores HTTP

  • HTTP1: IP → 192.168.1.40, HTTP activado
  • HTTP2: IP → 192.168.1.50, HTTP activado

📨 6️⃣ Configuración de Servidores de Email

Configurar Mail Service y crear usuarios de prueba:

  • Email1: 192.168.1.60 → usuario: alumno1@sitio1.com
  • Email2: 192.168.1.70 → usuario: alumno2@sitio2.com

🧑‍💻 7️⃣ Configuración de PCs

  • Obtener IP por DHCP
  • Probar navegación a:
    • http://www.sitio1.com
    • http://www.sitio2.com
  • Probar envío de correo desde Email Client

✅ 8️⃣ Comprobación final

  • Las PCs deben poder navegar en ambos sitios web
  • Las PCs deben poder enviar y recibir correos entre dominio 1 y 2
  • Debe resolverse correctamente cada dominio DNS
