🌐 Configuración de Adaptador de Red en VirtualBox

🧭 1. Ir a la configuración de red

  1. Abrir VirtualBox
  2. Seleccionar la máquina virtual (sin iniciar)
  3. Hacer clic en Configuración
  4. Ir a la pestaña Red

🔌 2. Elegir el tipo de adaptador

🌍 NAT

  • Tiene comunicación con la máquina host (computadora fisica)
  • Usa la conexión de tu PC
  • Tiene internet automáticamente
  • No es visible en la red

🖧 Adaptador Puente (Bridge)

  • Tiene comunicación con la máquina host (computadora fisica)
  • La VM actúa como una PC más
  • Recibe IP del router (DHCP)
  • Puede comunicarse con otras máquinas

🔒 Red Interna

  • Solo comunicación entre máquinas virtuales
  • No tiene internet

💻 Solo-Anfitrión (Host-Only)

  • Comunicación entre tu PC y la VM
  • Sin internet (por defecto)

⚙️ 3. Configurar detalles

  • Bridge: Elegir placa de red (WiFi o Ethernet)
  • Host-Only: DHCP automático o IP manual
  • NAT: Generalmente no requiere cambios

💻 4. Configuración dentro del sistema operativo

🪟 Windows

  • Panel de control → Red
  • Configurar IP automática (DHCP) o manual

🐧 Linux

  • Comando: ip a
  • Configurar desde Network Manager

Utilizar

Usar 2 adaptadores:
  • 🌍 Adaptador 1: Red Interna
  • 🔒 Adaptador 2: Red Interna
Ambos en el virtual box deben tener el mismo nombre, y luego en Windows server uno se va a llamar Ethernet y otro Red Interna.

🖥️ Ejercicio 1: Configuración Básica

🎯 Objetivo

Configurar parámetros básicos del servidor: nombre, escritorio remoto, red (DHCP y estática) y verificar configuración con comandos.

✏️ 1. Cambiar el nombre del servidor

  1. Ir a Inicio
  2. Click derecho en Este equipo → Propiedades
  3. Seleccionar Cambiar configuración
  4. Click en Cambiar
  5. Ingresar nuevo nombre (Ej: SERVER-01)
  6. 🔁 Reiniciar el equipo

🖥️ 2. Activar Escritorio Remoto

  1. Ir a Configuración
  2. Entrar en Sistema
  3. Seleccionar Escritorio remoto
  4. Activar la opción
  5. Permitir usuarios si es necesario

🌐 3. Configurar IP por DHCP

  1. Ir a Panel de control
  2. Red e Internet → Centro de redes
  3. Cambiarle el nombre por Internet
  4. Cambiar configuración del adaptador
  5. Click derecho en la red → Propiedades
  6. Seleccionar Protocolo IPv4
  7. Elegir:
    • ✔ Obtener IP automáticamente
    • ✔ Obtener DNS automáticamente

📌 4. Configurar IP Estática

  1. Cambiarle el nombre por LAN
  2. Ir nuevamente a IPv4
  3. Seleccionar Usar la siguiente dirección IP
  4. Completar:
    • IP: 192.168.1.2
    • Máscara: 255.255.255.0
    • Puerta de enlace: 192.168.1.1
    • DNS: 8.8.8.8

🔍 5. Verificar configuración con ipconfig

  1. Abrir CMD
  2. Ejecutar:

ipconfig

También se puede usar:

ipconfig /all

✅ Resultado esperado

  • Servidor con nombre configurado
  • Escritorio remoto habilitado
  • Red funcionando en DHCP
  • Red funcionando con IP estática
  • Verificación correcta con ipconfig

🏢 Ejercicio 2: Instalación de Active Directory

Active Directory en Windows Server

Active Directory (AD) es el servicio de directorio centralizado de Microsoft diseñado para Windows Server. Funciona como una base de datos segura que almacena, organiza y gestiona todos los recursos de una red empresarial (usuarios, contraseñas, computadoras e impresoras) y controla quién tiene permiso para acceder a ellos.

Componentes y Estructura

  • AD DS (Active Directory Domain Services): Servicio principal que almacena los datos y gestiona la comunicación entre usuarios y dominios.
  • Controlador de Dominio (Domain Controller): Servidor que ejecuta AD DS, autentica usuarios y aplica políticas de seguridad.
  • Objetos: Elementos individuales dentro de la red (usuarios, equipos, grupos, impresoras).
  • GPOs (Group Policy Objects): Reglas centralizadas que configuran lo que los usuarios pueden o no hacer en sus equipos.
  • Dominios, Árboles y Bosques: Jerarquías de organización lógica. El bosque es el contenedor principal de la infraestructura.

Funciones Principales

  • Autenticación: Comprueba la identidad de los usuarios mediante sus credenciales al iniciar sesión.
  • Autorización: Dicta a qué recursos, archivos o carpetas compartidas tiene acceso cada persona.
  • Gestión centralizada: Permite administrar cuentas, cambiar contraseñas y aplicar políticas en toda la red desde un solo lugar.

🏢 Ejemplo de empresa

La empresa tiene dos sucursales:

  • 📍 Ramos Mejía
  • 📍 San Justo

Dominio raiz (bosque) a crear:

empresa.com

🎯 Objetivo

Instalar Active Directory y crear el dominio empresa.com

⚙️ 1. Abrir el Administrador del Servidor

  1. Abrir Administrador del servidor
  2. Click en Agregar roles y características

⚙️ 2. Tipo de instalación

  1. Seleccionar Instalación basada en características o roles
  2. Click en Siguiente

⚙️ 3. Seleccionar servidor

  1. Elegir el servidor
  2. Click en Siguiente

⚙️ 4. Seleccionar rol

  1. Activar Servicios de dominio de Active Directory
  2. Aceptar agregar características
  3. Click en Siguiente

⚙️ 5. Instalar

  1. Click en Instalar
  2. Esperar que finalice

🏗️ 6. Promover a controlador de dominio

  1. Click en la bandera amarilla
  2. Seleccionar Promover este servidor a controlador de dominio
  3. Elegir Agregar un nuevo bosque
  4. Ingresar dominio: empresa.com
  5. Configurar contraseña
  6. Continuar con Siguiente hasta instalar

🔁 7. Reinicio del servidor (CLAVE)

  1. El sistema pedirá reiniciar
  2. Aceptar reinicio
  3. Esperar a que el servidor reinicie

Importante: Sin reiniciar, Active Directory no funcionará correctamente.

🔍 8. Verificación

  • Iniciar sesión como: EMPRESA\Administrador
  • Verificar funcionamiento del dominio

✅ Resultado esperado

  • Active Directory instalado
  • Dominio empresa.com funcionando
  • Servidor como controlador de dominio
  • Base para organizar sucursales (Ramos Mejía y San Justo)

🖥️ Ejercicio 3: Instalación y Configuración de DHCP

📌 ¿Qué es DHCP?

El DHCP (Dynamic Host Configuration Protocol) asigna automáticamente:

  • Dirección IP
  • Máscara de subred
  • Puerta de enlace
  • DNS

Evita configurar todo manualmente en cada PC 👍

⚙️ 1. Instalación del rol DHCP

  1. Abrir Administrador del servidor
  2. Click en Agregar roles y características
  3. Tipo de instalación:
    • 👉 Instalación basada en características o roles
  4. Seleccionar el servidor
  5. Marcar:
    • ✅ Servidor DHCP
  6. Siguiente → Instalar

🔐 2. Autorizar el servidor DHCP

  • Ir a la notificación ⚠️ en el Administrador del servidor
  • Click en Completar configuración de DHCP
  • Autorizar (si hay dominio)

✔️ Esto es necesario para que funcione en red

🌐 3. Crear un ámbito (Scope)

  1. Abrir Herramientas → DHCP
  2. Expandir el servidor
  3. Click derecho en IPv4 → Nuevo ámbito

Configuración típica:

  • Nombre: NombreEscuela-Año por ejemplo Tecnica10-26
  • Rango IP:
    • Inicio: 192.168.1.100
    • Fin: 192.168.1.200
  • Máscara: 255.255.255.0
  • Excluir IPs: (ej: router → 192.168.1.1)
  • Duración concesión: (default está bien)

🚪 4. Configurar opciones del ámbito

  • 003 Router (Gateway): 👉 192.168.1.1
  • 006 DNS: 👉 IP del servidor DNS o router

▶️ 5. Activar el ámbito

  • Click derecho en el ámbito
  • 👉 Activar

✔️ Ya empieza a repartir IPs automáticamente

💻 6. Configurar cliente (PC)

  1. Ir a Propiedades de red
  2. Seleccionar IPv4
  3. Marcar:
    • ✅ Obtener IP automáticamente
    • ✅ Obtener DNS automáticamente

Comandos en CMD

ipconfig /release
ipconfig /renew

🧪 7. Verificación

ipconfig
  • ✔️ IP dentro del rango configurado
  • ✔️ Gateway correcto
  • ✔️ DNS correcto

⚠️ Problemas comunes

  • ❌ IP 169.254.x.x → DHCP no responde
  • ❌ Ámbito no activado
  • ❌ Servidor no autorizado
  • ❌ Firewall bloqueando
  • ❌ Cable o red mal conectada

📘 Ejemplo completo

  • Red: 192.168.1.0/24
  • Servidor DHCP: 192.168.1.10
  • Router: 192.168.1.1
  • Rango DHCP: 192.168.1.100 - 192.168.1.200
